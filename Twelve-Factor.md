# 마이크로 서비스를 위한 설계 원칙 
```
* 클라우드에 배포 가능한 소프트웨어(SaaS) 를 구축하기 위한 설계원칙  // 12factor.net 참고
The twelve-factor app is a methodology for building software-as-a-service apps that

- 애노테이션을 사용해서 새로운 프로젝트에 참여하는 개발자의 시간과 비용을 최소화 시킬 수 있다.
- 운영 시스템의 계약(인터페이스 설계?) 을 명확하게 함으로써 새로운 컴포넌트를 운영환경에 이식하는 것을 극대화 시킬 수 있다.
- 이 설계 원칙은 최신 클라우드 플랫폼에 배포하기 적합하다.
- 운영환경과 개발환경 간의 간극을 최소화하고 지속적 배포를 위한 agility 를 극대화 한다. 
- 툴링, 아키텍처, 개발 관행? 을 크게 변경하지 않고 서비스를 확장 할 수 있다.
(And can scale up without significant changes to tooling, architecture, or development practices.)
```
